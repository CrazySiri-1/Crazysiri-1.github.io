## 早期的探索与挫折

在开源软件开发课程中，我开始了使用 GitHub Pages 搭建个人博客的尝试。

起初，我对 GitHub 仓库的命名规则不够了解，导致多次部署失败。经过反复检查文档和尝试，我意识到仓库名称必须与 GitHub 用户名完全一致，才能正确触发 Pages 服务。

随后，我在仓库中创建了 `_posts` 文件夹，并按照 Jekyll 的规范命名文章文件。通过观察 GitHub Actions 的运行日志，我逐渐理解了自动构建与部署的完整流程。

这次探索虽然遇到了一些波折，但也让我对开源工具链有了更具体的认识。
